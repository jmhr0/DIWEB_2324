Principios de Accesibilidad Web
Principio Criterio A Técnica Suficiente A Criterio A+ Técnica Suficiente A+ Criterio A++ Técnica Suficiente A++
Perceptible 1.1.1 Texto Alternativo para Imágenes Proporcionar texto alternativo descriptivo para imágenes utilizando el atributo "alt". 1.2.1 Multimedia Alternativo Proporcionar alternativas en texto o descripciones auditivas para contenido multimedia. 1.3.1 Info y Relaciones (prerequisito) Utilizar estructuras semánticas HTML para proporcionar información y relaciones adecuadas.
Operable 2.1.1 Accesibilidad del Teclado Permitir la navegación y la interacción mediante el uso exclusivo del teclado. 2.4.7 Enfoque Visible (prerequisito) Proporcionar indicadores visuales claros para elementos enfocados. 2.5.5 Nombre del Botón de Entrada (prerequisito) Utilizar etiquetas de formulario con textos descriptivos para los botones de entrada.
Comprensible 3.1.1 Idioma de la Página Indicar claramente el idioma principal de la página utilizando la etiqueta "lang". 3.2.1 Enfoque Visible (prerequisito) Proporcionar indicadores visuales claros para elementos enfocados. 3.3.4 Identificación de Errores (prerequisito) Proporcionar mensajes de error claros y sugerencias para corregir los errores de entrada del usuario.
Robusto 4.1.1 Análisis Sintáctico Utilizar código HTML semántico y válido para garantizar la compatibilidad con tecnologías de asistencia. 4.4.5 Análisis Sintáctico (prerequisito) Utilizar herramientas de validación para garantizar que el código cumpla con los estándares. 4.4.6 Encabezado de Sección Utilizar correctamente las etiquetas de encabezado para organizar y estructurar el contenido de la página.
